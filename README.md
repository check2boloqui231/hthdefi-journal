# hthdefi-journal

## 项目简介

`hthdefi-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为多个静态 HTML 页面的集中存放与版本管理工具。每个页面均为独立、自包含的 HTML 文件，可单独浏览或部署。

## 目录说明

```
hthdefi-journal/
├── README.md          # 本文件
├── pages/             # 存放所有独立 HTML 页面的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── assets/            # 公共资源文件（可选）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：核心目录，所有 HTML 页面按编号或其他规则存放于此，每个页面可独立访问。
- `assets/`：可选目录，用于存放多个页面共享的样式、脚本或图片资源，以保持页面结构整洁。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，包含必要的 CSS 与 JavaScript（如有），不依赖外部链接。
- 页面内容不指向任何特定网站或服务，仅作为信息展示或功能演示。
- 页面命名建议采用顺序编号或描述性名称，便于识别与管理。
- 提交新页面时，请确保文件无外部依赖，并放置于 `pages/` 目录下。

## 维护说明

- 本仓库主要用于个人或团队的页面归档，不提供在线服务或交互功能。
- 如需添加新页面，请直接在 `pages/` 目录下创建 HTML 文件，并更新本 README 中的目录列表（可选）。
- 若页面涉及敏感或临时内容，建议在文件名或注释中标注版本或日期。
- 仓库不对外提供直接访问链接，所有页面仅供存档或本地预览使用。
- 欢迎提交 Issue 或 Pull Request 以改进页面质量或结构，但请勿添加营销推广内容。
